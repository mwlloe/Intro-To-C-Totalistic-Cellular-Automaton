# Intro-To-C-Totalistic-Cellular-Automaton
this is project 1 from my intro to C (Programming Practicum) language. This introduces the use of similar coding structures/concepts, structs, typedefs, void functions, arrays, how to print, loops, global/const variables and more. This is the main(c) file. The purpose of this project is to provide real-world examples of cell characteristics...basically.

This code can be compiled to visualize a one-dimensional totalistic cellular automaton, which is essentially a fancy way of saying “a line of tiny decision-makers that update themselves based on their neighbors.” The world consists of 65 cells arranged in a row, and each cell can be in one of three states: 0, 1, or 2. For visualization, these are printed as a blank space, a dash (“-”), and a plus sign (“+”) respectively. Over a number of generations, the cells evolve according to a rule, producing patterns that can range from neat and symmetric to absolute chaos (depending on your rule choice and luck).

Each cell is represented by a struct that stores three values: its current state (status), the sum of itself and its immediate neighbors (localSum), and a running total (count) that tracks how “active” the cell has been across all generations. The local sum is the key driver of change here. Since each of the three cells involved can be 0, 1, or 2, the sum ranges from 0 to 6. This means there are exactly seven possible neighborhood outcomes, which keeps things simple but still interesting.

The rule that governs the automaton is provided by the user as an integer between 0 and 2186. This number is converted into base 3 and stored in an array of length 7. Each index in the array corresponds to one of the possible local sums (0 through 6), and the value at that index tells the program what the next state of a cell should be. In other words, instead of checking every possible pattern explicitly, the program just computes the sum and looks up the answer. It’s like a tiny decision table for each cell—efficient and slightly smug about it.

To keep the world continuous, the program treats the array as circular. This means the first and last cells are neighbors, so nothing falls off the edge of the universe. A helper function computes the local sums for all cells, and another function applies the rule to update every cell’s state simultaneously. While doing this, it also updates each cell’s cumulative count and keeps track of the total activity in the system for that generation.

In main, the user is prompted to enter a valid rule, the number of generations to simulate, and the initial value of the center cell (either 1 or 2). The world is then initialized so that all cells are zero except for the middle one, which acts as the seed. From there, the simulation runs for the specified number of generations. After each step, the program prints the current state of the world as a line of characters, along with the total sum of all cell values. Watching it evolve feels a bit like observing digital life; very simple organisms with surprisingly expressive behavior.

After all generations are complete, the program prints a summary showing how active each cell has been over time. This is displayed as two rows of digits representing the tens and ones places of each cell’s total count. Cells that never became active remain blank, which is a polite way of saying they contributed absolutely nothing to society.

Overall, the program is a neat example of how complex, sometimes mesmerizing patterns can emerge from very simple rules. Each cell only looks at its immediate neighbors and follows a tiny lookup table, yet the system as a whole can behave in ways that feel far more sophisticated than the code suggests. It’s a good reminder that even minimal logic, when repeated enough times, can get surprisingly interesting... kind of like recursion, but with more personality.
